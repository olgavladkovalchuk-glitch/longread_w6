Модуль 6: Финал

Упаковка, Деплой и Релиз

Как показать свой образовательный проект миру, внедрить его на платформу и подвести итоги обучения.

Часть 1: Публикация проекта

1

Заливка на GitHub Pages

Чтобы ваш код заработал в интернете, его нужно где-то разместить. GitHub — идеальный вариант для проектов. Создаем репозиторий, загружаем файлы и включаем Pages в настройках.

2

Вставка в Тильду (Блок Т123)

Когда проект лежит на GitHub, мы можем встроить его на лендинг в Тильде. Для этого используется HTML-тег <iframe>.

<iframe 
  src="https://твой-логин.github.io/твой-проект/index.html" 
  style="width:100%;height:100vh;border:none;display:block;">
</iframe>
<iframe>
Тег, который создает "окно" на вашей странице, через которое виден другой сайт.
src=".../index.html"
Самый важный атрибут. В конце ссылки обязательно допишите index.html!
style="..."
Убираем рамки и растягиваем проект на 100% высоты экрана пользователя (100vh).
3

Сборка SCORM-пакета (для LMS)

Если вы загружаете проект в полноценную LMS (Эквио, iSpring и др.), его нужно упаковать в ZIP-архив по стандарту SCORM.

Скопируй промпт для ИИ:
Напиши полный код для файла imsmanifest.xml (SCORM 1.2). У меня образовательный интерактивный лонгрид. Главный запускаемый файл называется index.html. Также в корне лежат папки [video, images]. Название курса: "Мой проект". Сделай код готовым к копированию.
⚠️ Важное правило архивации:
Когда будешь делать ZIP-архив, не архивируй саму папку целиком. Зайди внутрь папки, выдели все файлы и нажми «Добавить в архив».
